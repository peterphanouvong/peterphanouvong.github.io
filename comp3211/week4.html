Pipelined Processor (III)

Control hazards happen usually when there is a branch instruction. There is a delay in determining the outcome of the branch which, if handled with a stall, with hinder the speed of the pipelined processor.

Stall

We can stall and wait on every single branch to make sure its completed before moving on.

Prediction

Instead of having a constant amount of clock cycles given to completing the worst case branch, we can use prediction.

Static

Static prediction is predicting that the outcome of the branch will always be true(or false). If the prediction is correct, we save time. Otherwise if the prediction is wrong we have to do a flush to fix it. A flush means that all the instructions which were lined up to be executed next must be discarded. Overall though, the average amount of cycles spent on branching should decrease.

Dynamic

Dynamic prediction is predicting the outcome of the branch based off of past branch results (past results are held in a branch history table/prediction table).

1 Bit Prediction Scheme: The history bit is 1, then predict taken. If 0, predict not taken. If wrong, flip the history bit.
