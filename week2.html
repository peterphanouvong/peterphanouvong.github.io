Multi-Cycle Processor

The main problem with single cylce processors is its performance.

  • Each instruction that is run on a single cycle processor takes one clock cycle (CPI = 1)
  • The length of this clock cycle depends on the longest intruction
  • The longest instruction in lw which has to do the initial PC, Instruction Fetch, Registers, Mux and then it has the ALU execution, accessing Data Memory and write back.
  • Each instruction must now run on a clock cylce time dependent on the length of time it takes for lw to execute.

What is a Multi-Cycle Processor

The main things to note about the Multi-Cycle Processor are:

  • It has registers between the main stages of the processor
  • One clock cycle for each stage
  • Clock Cycle time dependent on the longest stage
  • CPI depends on type of instruction. (R-type uses IF, RR, EX, WB which would be 4 clock cylces)

Why is it better?

Since the CPI is smaller for instructions which require less stages, the average clock time for the instructions decreases (as opposed to a single cycle where all instructions are on clock cycle and execution time given by the longest instruction: lw).

overall latency is reduced, thus performance is increased.
