RESTful API Security

"Security provided by IT SYstems can be defined as the IT system's ability to being able to protect confidentiality and integrity of processed data, procide availability of the suste, and data, accountability for transactions processed, and assurance that the system will continue to perform to its design goals."

Least Privilege

Says that what the user can do is limited to the actions required to fulfil what they need to do. Nothing more, nothing less.

Fail-Safe Defaults

Default should be no authorisation. Then grant authorisation to methods. Not blacklist the ones the shouldn't have. Its whitelist the ones they are allowed to have.

Economy of Mechanism

Make it simple and easy to read. Less complexity = less chance of vulnerability

Complete Mediation

Authorisation required for every access

Open Design

Secure the system as if it is an open design. Rely on proper protectoion

Sparation of Privilege

Multiple keys is better than one key.

Least Common

If noone needs an action, dont provide it

Psychological Acceptibility

Ease of use, general user control flow should followe best security practices.

Defense in Depth

Control should be implemented in layers. If one layer fails, there will be a backup protection layer.

HTTPS (TLS)

https provides secure connection between client and server. Transfer layer security.

HTTP Basic Authentication

username and pword encoded in base 64. Not very secure as it can be decoded easily. Password is also sent repeatedly for each request.

Pretty bad idea...

Token Based Method

You login once, then the server gives you a token. The token is the only thing that is looked at to stay logged in.

Token is not stored in the system state, its self-contained token that stores it client-side. Tokens are expendable and are refreshed constantly.

No more password sent repeatedly and system remains stateless/

JSON Web Token

The structure of a JWT is the header(type), payload(data) and signature(encryption info).

All this data is encrypted using a hash and secret. Decoding only happens when both the sender and receiver know the secret.

API Key

When you sign up for the service, the API issues the user an API Key. It tells the API whos is using the service.

Helps manage rate limiting, get analytics,, issue further access tokens.

They require a separate platform to manage the keys otherwise it becomes a vulnerability if its managed within the system

OAuth

Solves the problem of a third party trying to access client information without sharing vital credentials.

It's essentially the logging in with facebook

  1. authiration request to user
  2. authorisation granted with a token
  3. give the token to the main server
  4. server reponds with an access token
  5. request for customer data with the accces token
  6. receive customer data

For this to occur, the third party service must be registed with the main server. Authorisation of the data is done with OAuth.

Machine Learning

There are many machine learning techniques used to analyse data...

All these techniques follow a similar process:

  1. Prepare the data
  2. Define and Initalize the model
  3. Train your model
  4. Validate the model
  5. Use the model - explore or deploy as a web service
  6. Update and revalidate.

When inputting data, there is a common flow...

Machine Learning can be categorised into 3 categories:

  • supervised
  • unsupervise
  • reinforcement

Questions which ML can answer

  • Classification: A or B?
  • Clustering: divide into groups of similar characterstics
  • Regression: How many/How much over time? Mapping to a function
  • Anomoly: Which on is the odd one out?
  • Reinforcement: What do we do now?
