Classification

Classifiers categorise data with labels

Give the classifer training data and then give it an answer(label)

Then once the classifier has been trained with the training data and solution, you can then give it new input data and it will try to classify it into categories.

  • Lets say you have data (colour, weight, size) on fruits: apple, banana, orange
  • Given the all the data with corresponding labels you can train the classifier to then predict what the fruit is, given some colour, wieght and size.

KNN is a type of classifier that uses the principle of K-nearest neighbours

  • It is considered to be NON-PARAMETRIC and INSTANCED BASED.
  • this means that it will have no prexisting assumptions about the data and itll memorise the training instances as knowledge to predict the class of new inputs.

Requirements

The KNN Classifier has 4 requirements:

  1. distance metric: how to measure the closeness of 2 data points
  2. how many neighbours: how many data points to consider
  3. Weighting function: the weight of the data point (usually dependent on the distance)
  4. Aggregate function: how to collate the weights of each datapoint

All these factors alter and adjust how the data points are classified. You can change these factors as you like to better suit the data that you have.

In order to train the classifier you need to first have a dataset with answers and that.

  • What you can do is partition the dataset so that a percentage of the dataset is going in as the training set, and then have some leftover to use as input.
  • Once the dataset is trained, you can then use the leftover input and check it against the actual values. From there you can further adjust the settings to get a better fit.
  • train_x : training input
  • train_y: training output
  • test_x: testing input
  • test_y: desired testing output, used to check against the actual output of the classifier.

The aim of using machine learning like this is to have a general model that you can use to classify new inputs.

  • Getting a good model which can be applied to new inputs is call Generalisation
  • Overfitting: is when you're classifier only really works for your dataset. Its too fitted to the data its been given. This is usually because the number of neighbours is too small
  • Underfitting: is when the classifier doesn't really work at all because its too general. Usually when too many neighbours are taken into account.

Choosing an appropriate value for K

large K can make our dataset quite general, but too large and its not useful. It might be considering points which aren't true neighbours.

A general rule of thumb is to not go greater than the sqrt of k.

When to consider

  • When there aren't many columns/attributes to consider
  • Have lots of data

Advantages

  • Fast
  • Can learn complex target functions
  • No loss of data

Disadvantages

  • Slow at query
  • sensitive: if you have a noisy dataset then the bad neighbours can really mess things up
  • Bigger the dataset, more impact on performance

Decision Tree

Pretty old Machine Learning algorithm.

Very intuitive and easy to implement

When to consider

  • Data available in attribute value pairs
  • discrete values
  • Missing data
  • possible noise
  • eg. medical diagnosis, risk analysis

Essentially you take in some data and then you create a decision tree based off of the data that you have, then you can introduce new data to be parsed into the decision tree which will then give you some output.

Simpler and smaller is better

It may be human readable, but not follow human logic
